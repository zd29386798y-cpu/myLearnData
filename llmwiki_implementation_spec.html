LLMへの実装計画指示書
前提条件
  • OS: Windows 10/11 (64bit)
  • RAM: 8〜16GB（GPU/VRAM なし、CPU推論）
  • 使用シーン: A=事務所LANあり / B=完全オフライン / C=計算機室LANあり
  • チーム人数: 2〜10人
  • 制御システム数: 最大20（Workspaceで分離）
  • すべて無料・OSSツールのみ使用
フェーズ 1 llama.cpp セットアップ
  1. llama.cpp の Windows用ビルド済みバイナリ（llama-b****-bin-win-cpu-x64.zip）を GitHub Releases からダウンロード
  2. Hugging Face から Qwen2.5-7B-Instruct-Q4_K_M.gguf を取得し models/ フォルダに配置
  3. 以下のコマンドでサーバーモード起動（OpenAI互換 API を localhost:8080 で公開）：
    llama-server.exe -m models/Qwen2.5-7B-Instruct-Q4_K_M.gguf --host 0.0.0.0 --port 8080 --ctx-size 4096 -ngl 0
  4. 起動バッチファイル（start_llm.bat）を作成し、ダブルクリックで起動できるようにする
  5. ブラウザで http://localhost:8080 にアクセスし動作確認
-ngl 0 はGPUオフロードなし（CPU専用）の指定。RAM 8GBの場合は --ctx-size 2048 に下げること。
フェーズ 2 AnythingLLM セットアップ（RAG + チャットUI）
  1. AnythingLLM Desktop版インストーラーを公式サイト（useanything.com）からダウンロード・インストール
  2. 初回起動後、LLMプロバイダーを「Custom OpenAI」に設定し http://localhost:8080/v1 を指定
  3. 埋め込みモデルを「AnythingLLM Native Embedder」（nomic-embed-text）に設定
  4. 制御システムごとにWorkspaceを作成（最大20個）。命名規則例：SYS01_ボイラー制御
  5. 各Workspaceに対応する既存資料（PDF/Word/Excel）をアップロードしてRAGインデックスを構築
  6. チャット動作確認：資料を参照した回答が返ることを確認
Word/ExcelはPDF変換してからアップロードすると精度が上がる。変換はLibreOffice（無料）で一括変換可能。
フェーズ 3 Obsidian wiki セットアップ
  1. Obsidian を公式サイトからダウンロード・インストール（無料）
  2. Vaultのフォルダ構成を以下のように作成：
    LLMwiki/
    ├─ SYS01_ボイラー制御/
    ├─ SYS02_冷却系統/
    └─ _templates/ （回答保存テンプレート）
  3. テンプレートファイル _templates/回答メモ.md を作成（日付・質問・回答・タグフィールドを含む）
  4. Obsidian の「Templates」コアプラグインを有効化し、テンプレートフォルダを設定
フェーズ 4 回答→wiki 自動保存スクリプト（AutoHotkey）
  1. AutoHotkey v2 を公式サイトからダウンロード・インストール（無料）
  2. 以下の動作をするスクリプト（save_to_wiki.ahk）を作成：
    • ホットキー（例：Ctrl+Shift+S）を押す
    • クリップボードの内容（AnythingLLMでコピーした回答）を取得
    • ファイル名ダイアログを表示（保存先Workspace・タイトルを入力）
    • テンプレートに日付・内容を埋め込み、指定Workspaceフォルダへ .md ファイルとして保存
  3. スクリプトをスタートアップに登録し、常駐させる
  4. 動作テスト：AnythingLLMで回答を選択→コピー→Ctrl+Shift+S→保存確認
手動トリガー（ホットキー）で起動するため、保存したい回答だけを選別できる。全回答の自動保存よりノイズが少ない。
フェーズ 5 同期設定（シーンA/C）
  1. Syncthing を両PCにインストール（公式サイト、無料）
  2. WebUI（localhost:8384）で相互にデバイスを追加し、LLMwiki/ フォルダを共有設定
  3. LAN内での自動同期を確認（ファイル変更後、数秒で反映）
  4. AnythingLLMのRAGデータ（Vectorストア）の同期方法を決定：
    ・方法A：各PC独立でインデックスを保持（資料追加時は両PCで実施）
    ・方法B：AnythingLLMのデータフォルダごとSyncthingで同期（推奨）
AnythingLLM Desktopのデータフォルダはデフォルト %APPDATA%\anythingllm-desktop。このフォルダをSyncthingの共有対象に追加するとRAGデータも同期できる。ただし同時起動は避けること。
フェーズ 6 シーンB オフライン持出し手順（文書化）
  1. 持出し用USBに以下をコピーするバッチ（export_for_offline.bat）を作成：
    • llama.cpp フォルダ（バイナリ＋モデル）
    • AnythingLLM インストーラー＋データフォルダ
    • LLMwiki/（Obsidian Vault）
    • save_to_wiki.ahk（保存スクリプト）
  2. ノートPCへのインストール手順書（README.md）をwikiに保存
  3. 現場作業後、更新された LLMwiki/ を事務所PCにUSBで持ち帰りSyncthingで取り込む手順を文書化
追加オプション チーム運用ルール（文書化推奨）
  • Workspaceの命名規則（例：SYS番号_システム名）
  • 資料追加時の手順（PDF変換→AnythingLLMアップロード→インデックス確認）
  • wikiメモの保存タグ規則（例：#設計 #トラブル #手順）
  • シーンBからの帰還同期タイミング（作業終了後24時間以内など）
