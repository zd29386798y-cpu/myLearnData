LLMwiki 要件定義 v2

目的
目的 制御システム毎のナレッジ蓄積と調査効率化
  • 設計・トラブル時に既存資料をAIが参照し、要約・回答を生成する
  • 回答をMarkdownファイルとしてwikiに保存・編集・蓄積できる
  • 制御システム単位でwikiを分離管理（フォルダ構成で対応）
使用シーン（制約整理）
🏢 シーン A — 事務所

ローカルLAN あり
PC複数台で共有アクセス可
サーバー役PCを1台立てられる

⚙️ シーン B — 計算機室（持込PC）

完全オフライン
ネットワーク接続なし
ノートPC単体で完結が必要

🖥️ シーン C — 計算機室（固定端末）

ローカルLAN あり
端末に同構成を構築
シーンAと同じ構成で対応可

設計方針： A・C はサーバー機1台 + クライアントWebブラウザ構成。B はノートPC 1台で完全自己完結する「ポータブル構成」を用意。共通のMDファイル構成にしておけば、Bで作成した資料をAに持ち込んで反映できる。
① ローカルLLM構成
確定 llama.cpp + Webサーバーモード
  • llama.cpp の --server モードでOpenAI互換APIとして起動
  • RAGやUI側から http://localhost:8080 で呼び出す構成
推奨モデル（8〜16GB RAM想定）：
RAM 8GB → Qwen2.5-7B-Instruct Q4_K_M（4.5GB、日本語◎）
RAM 16GB → Mistral-7B-Instruct v0.3 Q5_K_M（5.1GB）または Llama-3.1-8B Q4_K_M（5.0GB）
※ gguf形式でHugging Faceから無料取得
③ RAG構成（推奨）
推奨 AnythingLLM — オールインワンRAGツール
  • PDF・Word・Excel・Markdown・テキストを自動でロード・チャンク化
  • 埋め込みモデル（nomic-embed-text）をローカルで実行、外部送信なし
  • ベクターDB（ChromaDB）内蔵、追加セットアップ不要
  • チャットUI内蔵、llama.cppのOpenAI互換APIと直接接続可能
  • Workspace（≒制御システム毎のwiki）単位でRAGの参照範囲を分離できる
AnythingLLM はWindows版インストーラーあり。Dockerなしで動作可能なため、シーンBのオフライン環境にも対応しやすい。
既存資料 (PDF/Word/Excel) → AnythingLLM（ローダー・チャンク化） → ChromaDB（ベクター検索） → llama.cpp（回答生成） → チャットUI（表示・保存）
② wiki保存・編集（MDファイル前提）
候補保存形式チーム共有オフライン推奨度
Obsidian + Syncthing MDファイル Syncthing で自動同期 ◎ 完全対応 ◎ 推奨
Outline（自己ホスト） MD互換DB ◎ Webブラウザで共有 × サーバー必要 △ シーンBに非対応
MkDocs（静的） MDファイル ファイル共有のみ ◎ 完全対応 △ 編集UIなし
推奨：Obsidian + Syncthing
MDファイルをそのまま編集・閲覧。Syncthingでシーン A↔C 間のLAN同期、シーンBはUSB経由でファイルコピーに対応。AnythingLLMの回答をコピー→Obsidianに貼付けという運用が最もシンプル。

役割ツール備考

残り未決定事項（実装計画前に確認）
  • PCスペック詳細（GPUあり/なし、VRAM量）→ モデルのquant設定に影響
  • シーンA/CのサーバーはノートPC兼用か、専用機か
  • AnythingLLMの「回答→Obsidian保存」は手動コピーで許容か、自動化（スクリプト）が必要か
  • 制御システムの数（Workspaceの分割数に影響）
